# Web Design Guidelines - Implementation Instructions

## Skill Purpose
Review UI/web code for compliance with Web Interface Guidelines using automated analysis.

## Execution Flow

### Step 1: Initialize
- Parse user request to identify which files to review
- If not specified, ask user for specific files or patterns (e.g., "*.tsx", "src/components/")

### Step 2: Fetch Guidelines
- Use `fetch_webpage` to retrieve guidelines from: https://raw.githubusercontent.com/vercel-labs/web-interface-guidelines/main/command.md
- Parse the fetched content to extract all applicable rules
- Categorize rules by type (accessibility, HTML, styling, etc.)

### Step 3: Analyze Files
- Use `file_search` to find matching files if a pattern is provided
- Use `read_file` to examine each file's contents
- Cross-reference code against guideline rules
- Note line numbers and specific violations

### Step 4: Compile Findings
- Organize findings by:
  - Severity (critical, warning, info)
  - Category (accessibility, semantics, styling, etc.)
  - File
- Include specific line references and code snippets

### Step 5: Provide Recommendations
- For each finding, suggest concrete improvements
- Provide code examples when helpful
- Prioritize issues by impact on user experience

## Key Rules to Check For

### Accessibility
- [ ] WCAG 2.1 AA compliance
- [ ] Proper ARIA labels and roles
- [ ] Keyboard navigation support
- [ ] Color contrast ratios (4.5:1 for text)
- [ ] Alt text for images

### Semantic HTML
- [ ] Proper heading hierarchy (h1-h6)
- [ ] Semantic elements (nav, main, article, etc.)
- [ ] Form labels properly associated
- [ ] Link text is descriptive

### Responsive Design
- [ ] Mobile-first approach
- [ ] Proper viewport meta tags
- [ ] Flexible layouts
- [ ] Touch-friendly click targets (44x44px minimum)

### Performance
- [ ] Image optimization
- [ ] CSS/JS bundling
- [ ] Lazy loading implementation
- [ ] Unnecessary dependencies

## Output Format

```markdown
# Web Design Guidelines Review

## Summary
- Total Issues Found: X
- Critical: X | Warnings: X | Info: X

## Findings by File

### [path/to/file.tsx]

**[line:45] CRITICAL - Accessibility Issue**
- Rule: Images must have alt text
- Current: `<img src="logo.png" />`
- Fix: `<img src="logo.png" alt="Company logo" />`

**[line:78] WARNING - HTML Semantics**
- Rule: Use semantic heading hierarchy
- Current: `<h3>Main Title</h3>`
- Fix: `<h1>Main Title</h1>`

## Recommendations
- [List prioritized improvements]
- [Quick wins first]
- [Long-term improvements]
```

## Tool Usage

| Task | Tool | How |
|------|------|-----|
| Fetch guidelines | `fetch_webpage` | Get latest rules from GitHub URL |
| Find files | `file_search` | Use glob patterns to locate files |
| Read files | `read_file` | Read specific files in batches |
| Search patterns | `grep_search` | Find specific code patterns |
| Search semantically | `semantic_search` | Find related components/functions |

## Best Practices

1. **Batch file reads** - Read multiple files in parallel when possible
2. **Fresh guidelines** - Always fetch the latest version before review
3. **Prioritize impact** - Focus on accessibility and UX issues first
4. **Be specific** - Always reference line numbers and code snippets
5. **Provide fixes** - Don't just report problems, suggest solutions
6. **Group findings** - Organize by severity and category for clarity

## Common Patterns to Check

```javascript
// ❌ Bad
<img src="avatar.jpg" />
<div onClick={handleClick}>Click me</div>
<h3>Main heading</h3>
<input type="text" />

// ✅ Good
<img src="avatar.jpg" alt="User avatar" />
<button onClick={handleClick}>Click me</button>
<h1>Main heading</h1>
<label><input type="text" placeholder="Search" /></label>
```
